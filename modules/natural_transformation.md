## Mathematical Foundation

**自然变换 (Natural Transformation)**：
给定两个函子 F, G: C → D，自然变换 α: F ⇒ G 是一族态射：
α_X: F(X) → G(X)，满足交换图条件（naturality square）

**商业映射**：
- F = 旧策略/旧映射逻辑
- G = 新策略/新映射逻辑
- α = 从旧到新的过渡方案
- 交换图 = 策略切换过程中保持核心关系不变

## Trigger

### 自动触发
用户输入包含以下关键词或语境：
- "市场环境变了"、"行业转型"、"政策调整"
- "原来的方法不管用了"、"策略失效"
- "如何过渡"、"如何平滑切换"
- "既要...又要..."（保持旧优势 + 获得新能力）

### 手动触发
命令：`/morphism-pivot`

## Logic

### Step 1: 识别环境变化 (Category C 的改变)
定义变化前后两个环境状态 C₁ → C₂：
- 市场结构变化（集中度、进入壁垒）
- 技术范式变化（旧技术 → 新技术）
- 政策/监管变化（合规要求、补贴退坡）
- 消费者行为变化（需求迁移）

### Step 2: 评估旧函子 F 的适用性
- 哪些映射关系 F(Oₐ) = Oᵦ 仍然有效？
- 哪些因为环境变化而失效？
- 失效的根本原因是结构破坏还是参数调整？

### Step 3: 构造新函子 G
在不改变 Domain A 和 Domain B 的前提下，重新定义映射规则：
- G(Oₐ) = Oᵦ'（可能映射到 B 中不同的对象）
- G(Mₐ) = Mᵦ'（关系映射方式改变）

### Step 4: 寻找自然变换 α
构造从 F 到 G 的过渡方案，满足：
- **对象层面的连续性**：每个 Oₐ 都有明确的 F(Oₐ) → G(Oₐ) 路径
- **关系层面的相容性**：原有 Morphism 在新旧映射中保持逻辑一致
- **最小破坏原则**：尽量保留 F 中仍然有效的部分

### Step 5: 验证交换图
确保以下方块可交换：
```
F(Oₐ) --α_Oₐ--> G(Oₐ)
   |                  |
F(f)               G(f)
   |                  |
   v                  v
F(Oᵦ) --α_Oᵦ--> G(Oᵦ)
```
商业含义：无论走"旧路"还是"新路"，最终到达的 Domain B 结论应该一致（或兼容）
